# Session — grapevine V1.7 implementation

**Date:** 2026-06-11 **Branch:** `feat/grapevine-v1.7` → `develop`

## What was built

Grapevine V1.7 — **the human as a first-class participant** in a channel. Built
in chapters on the branch:

1. **Phase 0 — Alpine port.** `watch.html` converted from ~555 lines of
   hand-synced vanilla JS to a single idiomatic Alpine `x-data` component (CDN +
   SRI, no build step). Behavior-preserving; verified against an equivalence
   checklist with Playwright before any feature work.
2. **Human as participant** — per-HOME `alias` config + `GET /identity`; a
   `human` presence flag (`who.humans`, `tail --human`); join/lurk from the
   watch; compose box.
3. **Threading** — `in_reply_to` (daemon passthrough + `send --in-reply-to` +
   watch reply UI, quoted/indented render).
4. **Archive vs close** — sidecar-marker read-only retire (`archive`/
   `unarchive`); sends rejected, history kept, name locked from re-open.
5. **Lurk model refinements (from the soak)** — default-lurk, explicit join
   **persisted per-channel** in localStorage, and **truly-invisible lurk**
   (`?lurk=1` / `tail --lurk`, excluded from every presence count).

Intentionally deferred (proposal-sanctioned): direct/`@mention` messages,
cross-channel `announce`, `kind:"correction"`, and a debounced presence/join
event (the last newly captured in the backlog).

## Process

- **Subagent-light, verification-heavy:** each chapter implemented directly,
  unit-tested (`bun test`), and verified live with Playwright.
- **Live soak** (`soak-findings.md`) — human in the browser + agent (`tycho`) on
  the CLI. Caught two emergent bugs: rejoin-on-channel-switch (→ default-lurk
  - persisted mode) and the lurker count-badge (→ `visibleSubs` at the missed
    `listChannels`/`open` sites). Both fixed + regression-tested mid-soak.
- **Ward** — drift check, decay-ledger bump, scenario captured
  (`grimoire/scenarios/2026-06-11-default-to-the-passive-state.md`), banner
  V1.6.7→V1.7, smoke test.
- **Cold fresh-agent test**
  (`grimoire/fresh-agent/2026-06-11-grapevine-v1.7-findings.md`) — every V1.7
  behavior matched; navigational SKILL gaps fixed (empty
  `${CLAUDE_PLUGIN_ROOT}`, `~/.grapevine` vs `$GRAPEVINE_HOME`, version-label
  confusion, archive-reject output shape).
- **Finalize-branch dual code review** — both "with fixes." Fixed: `--human`/
  `--lurk` missing from `BOOLEAN_FLAGS` (flag-order bug, confirmed live);
  `unarchive` swallowing an unlink failure; `msgById` O(n²); a `/presence` test
  gap; stale daemon comments.

## Key decisions

- **Stack: Alpine, not React** — grapevine stays in the lightweight (no-build)
  branch of the spell-surface threshold rule; React is reserved for
  glamour-class surfaces. (Memory: `spell-surface-stack`.)
- **Default to the passive state** — a soak-driven reversal of the initial
  "default-represented" call: viewing ≠ participating; default lurk, make
  joining explicit and remembered. Captured as a scenario.
- **Mode/alias live in localStorage**, presence in daemon memory, archive on
  disk, messages in JSONL — each piece of state in the right place.
