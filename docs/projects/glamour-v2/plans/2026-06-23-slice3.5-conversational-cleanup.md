# Slice 3.5 — Conversational Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Strip action-oriented chrome from the glamour-v2 surface and replace
it with implicit-presence affordances, so intent flows from the conversation and
direct manipulation rather than explicit buttons/toggles.

**Architecture:** glamour-v2 is a Bun daemon + React 19 surface (bundled by Bun,
Tailwind v4 CSS-first) over one shared `surface/state/types.ts` contract
(WebSocket full-state to the browser, SSE `/events` to the agent, HTTP
`POST /cmd`). This pass is mostly _subtractive_: it removes two client→server
imperatives' worth of UI (`generate`, the Add button) and one contract member
(`FeedbackMode`/`Message.mode`), adds two small composer affordances (Send
button, resizable textarea), one client-only liveness indicator, and fixes a
layout-containment bug plus a responsive-density gap.

**Tech Stack:** Bun, React 19, TypeScript, Tailwind v4 (CSS-first via
`surface/styles.css`), biome (format/lint), `bun test`.

**Source of truth for this work:**
`docs/projects/glamour-v2/sessions/2026-06-23-slices1-3-dogfood-feedback.md`
(the dogfood punch-list + the design principle this pass commits to).

## Global Constraints

- **Branch:** all work lands on `feat/glamour-v2` (every glamour-v2 slice is
  held there until the post-Slice-4 cutover). Do NOT branch, merge, or push.
- **Runtime:** Bun. Run tests with `bun test` from repo root. Never
  `npm`/`node`/`jest`.
- **Format before commit:** `bunx biome check --write <changed .ts/.tsx>`;
  prettier for any `.md`. (House rule: biome owns import-sort — do not run
  prettier on `.ts/.tsx`.)
- **Real buttons:** every clickable is a real `<button>` — never a pointer-only
  `<div>` with a click handler, never a `biome-ignore` for
  `noStaticElementInteractions`.
- **Contract invariant must stay green:** `tests/types.test.ts` "every
  imperative client message has an agent event type" — any browser message that
  is not a pure board move must be representable as an agent event. Removing
  `generate` (Task 1) means removing it from BOTH `ClientToServer` and
  `AGENT_EVENT_TYPES` together.
- **Design principle (governs ambiguity):** strip action-chrome; let intent flow
  from conversation + direct manipulation; controls live where the action
  visually is; as controls go away, the interface carries more implicit
  presence.
- **Commit story-chapters:** one self-contained commit per task,
  conventional-commit style, scope `glamour`. Trailer:
  `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Verification reality:** contract/logic changes have `bun test` guards. Pure
  CSS/JSX changes (Send button, resizable textarea, scroll containment,
  responsive columns, the indicator's visuals) are NOT unit-tested here — they
  are verified visually against a live session and the bundler build. Each such
  task states its exact visual check.
