# Scenarios — captured judgments

> _"When the familiar reaches a conclusion the mage disagrees with, capture the
> scenario: the mage explains the reasoning, and what's distilled is not the fix
> but the judgment that produced it."_

This is the **theoretical half** of the craft loop (the empirical half is
`../fresh-agent/`). It's apprenticeship, not instruction — you transmit the
judgment that _generates_ the rules, because that's the only thing that
generalizes to cases you haven't hit.

## When to capture

- The familiar concluded X; the mage wanted Y. The gap is a judgment worth
  keeping.
- A house-style rule was added, changed, or repealed — the scenario is its
  **repeal criterion** (Chesterton's fence with the builder's note).
- Two principles conflicted and something arbitrated. Capture how.

## What a scenario is for

1. **Distil the judgment**, not the fix. "Why did the mage think that?" — the
   reasoning that produced the conclusion, not the conclusion.
2. **Bind it to a rule.** If it spawned or changed a `house-style.md` rule, name
   that rule. The scenario is now the rule's repeal criterion.
3. **Reinforce.** Logging a scenario that re-walks an existing rule bumps that
   rule's date in `../decay-ledger.md` — that's how rules stay alive.

One file per scenario. Use `TEMPLATE.md`. Name them `YYYY-MM-DD-short-slug.md`.
