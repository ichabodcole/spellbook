# Start minimal; subtract before testing

## The situation

The grimoire and the authoring skills were being written with a lot of upfront
explicitness — rules, caveats, and detail added as the drafts grew.

## What the familiar concluded

More detail is safer and more complete: spell out the conventions, the edge
cases, the warnings, so nothing is left to chance.

## What the mage wanted instead

Agents **over-specify by default** when authoring skills — they just did the
work, so their hot context leaks onto the page as detail that _feels_ essential
but isn't. It's not only the specific traps (defensive negatives, restating the
discoverable) — it's general bloat. The first defense against rule accretion is
not adding in the first place. So: write the draft, make a **subtraction pass**
(cut to the least-explicit version you think works), _then_ fresh-agent test,
and add back only what the cold agent stumbles on. Framed as a **reflection lens
applied after writing**, not a constraint while writing.

## The distilled judgment

Treat over-specification as the _default failure mode_ of agent-authored
instructional content. Subtract before you test; let the cold-agent signal
restore only what's load-bearing. A line never written never has to decay.

## Binding

- **Rule affected:** spawned the new rule "Start minimal; subtract before you
  test"; added the subtraction pass to `inscribe` phase 5 and
  `fresh-agent/README` ("Subtract first").
- **Repeal criterion:** as models improve and cold agents stumble less, the
  subtraction pass can grow more aggressive (a temporal edge), not be removed.
